# RWS 二自由度云台跟踪系统 — 硬件集成指南

## 系统概述

RWS（Robot Weapon Station）是一个视觉跟踪 + 云台控制系统，用于自动跟踪和瞄准目标。

### 核心组件

1. **二自由度云台**（2-DOF Gimbal）
   - Yaw 轴：水平旋转（-160° ~ +160°）
   - Pitch 轴：垂直旋转（-45° ~ +75°）

2. **摄像头**
   - 固定在云台上，随云台转动
   - 推荐分辨率：1280x720 或更高
   - 帧率：30 FPS 以上

3. **控制器**
   - 运行 RWS 软件的计算机
   - 推荐：带独立显卡（用于 YOLO 推理）

## 硬件选型建议

### 方案 A：入门测试（500-1500 元）

**云台：**
- 淘宝 DIY 云台套件（搜索"二自由度云台 舵机"）
- 包含：2 个高扭矩舵机（20kg·cm 以上）+ 铝合金支架
- 控制方式：PWM 信号（Arduino / 树莓派控制）

**摄像头：**
- 罗技 C920 / C922（USB 摄像头，1080p）
- 或树莓派 Camera Module V2

**控制器：**
- 树莓派 4B（4GB / 8GB）+ USB 摄像头
- 或笔记本电脑 + USB 摄像头

**优点：** 成本低，易于调试
**缺点：** 舵机响应慢，精度一般

---

### 方案 B：专业级（3000-8000 元）

**云台：**
- Tarot T-2D（航拍云台，去掉 Roll 轴）
- DJI Ronin-S（手持稳定器改装）
- 自制无刷云台（BGC 控制器 + 无刷电机）

**摄像头：**
- 工业相机（如 FLIR Blackfly S，GigE 接口）
- 或高端 USB 相机（如 ELP 1080p 120fps）

**控制器：**
- NVIDIA Jetson Xavier NX / Orin Nano
- 或 Intel NUC + RTX 3060

**优点：** 响应快，精度高，适合实际应用
**缺点：** 成本较高，需要硬件调试经验

---

### 方案 C：机器狗集成（已有机器狗）

如果已有机器狗（如宇树 Go1/Go2、波士顿动力 Spot），可以：

1. 在机器狗背部安装云台
2. 使用机器狗的 IMU 数据进行体运动补偿
3. 通过机器狗 SDK 获取实时姿态（roll/pitch/yaw + 角速度）

RWS 已支持体运动补偿，只需实现 `BodyMotionProvider` 接口即可接入。

## 摄像头安装位置

### 标准安装（推荐）

```
        [摄像头]
            |
        [Pitch轴]
            |
        [Yaw轴]
            |
        [底座/机器人]
```

摄像头固定在 Pitch 轴上，随云台一起转动。

### 为什么摄像头要随云台转动？

1. **保持目标在视野中心** — 云台转动时目标始终在画面中心附近，减少视野边缘畸变影响。
2. **扩大跟踪范围** — 云台可旋转 ±160°，覆盖几乎全方位；固定摄像头仅覆盖有限视野（如 60°）。
3. **减少图像抖动** — 云台稳定器可以抵消机器人运动引起的抖动，提高检测和跟踪精度。

## 快速测试步骤

### 1. 仿真测试（无需硬件）

运行 `run_demo.py` 即为仿真模式，可以验证：
- 目标检测和跟踪逻辑
- PID 控制器参数
- 状态机切换
- 实时仪表盘显示

### 2. 摄像头测试（需要 USB 摄像头）

```bash
python run_yolo_cam.py
```

这会打开摄像头、运行 YOLO 检测、显示跟踪结果。云台使用仿真驱动，不会真正转动。

### 3. 真实云台测试（需要硬件）

当有真实云台后，需要：

1. **实现串口驱动** — 在 `src/rws_tracking/hardware/` 下创建 `serial_driver.py`，实现 `GimbalDriver` Protocol，通过串口发送命令。
2. **替换 Pipeline 配置** — 将 `SimulatedGimbalDriver` 替换为 `SerialGimbalDriver`，配置串口参数。
3. **标定相机内参** — 使用 OpenCV 棋盘格标定，更新 `config.yaml` 中的 `camera` 参数。

## 常见问题

### Q: 没有云台能测试吗？

可以。使用仿真模式（`run_demo.py`）或摄像头模式（`run_yolo_cam.py`），系统会模拟云台运动。

### Q: 摄像头必须固定在云台上吗？

是的。如果摄像头固定不动，云台转动后目标会离开视野，无法持续跟踪。

### Q: 支持哪些云台协议？

目前需要自行实现驱动。常见协议：
- PWM（舵机）
- UART / 串口（自定义协议）
- CAN（工业云台）
- MAVLink（航拍云台）

### Q: 如何提高跟踪精度？

1. 标定相机内参（消除畸变）
2. 调整 PID 参数（降低超调）
3. 启用弹道补偿（远距离目标）
4. 启用自适应 PID（不同距离自动调整增益）

### Q: 支持多目标同时跟踪吗？

当前版本是单目标跟踪（选择最优目标）。多云台协同跟踪需扩展架构，详见 [TODO.md](TODO.md)。
