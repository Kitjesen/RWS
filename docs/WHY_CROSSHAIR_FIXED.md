# RWS 系统工作原理 - 为什么准心不动？

## 🎯 核心概念

### 准心 = 云台瞄准方向

在 RWS 系统中：
- **准心（红色十字）** = 云台当前指向的方向
- **摄像头固定在云台上**，随云台一起转动
- **准心永远在画面中心**，因为它代表"云台看向哪里"

这就像你拿着相机拍照：
- 相机镜头的中心 = 准心
- 无论你怎么转动相机，镜头中心永远在画面正中间
- 但相机拍到的**内容**会随着你的转动而变化

---

## 🔄 系统工作流程

### 1. 目标检测
```
摄像头拍摄 → YOLO 检测 → 发现目标（人、车辆等）
```

### 2. 计算误差
```
目标位置：(x=800, y=400)
画面中心：(x=640, y=360)
误差：目标偏右上方
```

### 3. PID 控制
```
误差 → PID 计算 → 输出云台转动速率
例如：Yaw +15 dps（向右转）, Pitch +5 dps（向上转）
```

### 4. 云台转动
```
云台向右上方转动 → 摄像头跟着转 → 目标逐渐回到画面中心
```

### 5. 锁定目标
```
当误差 < 0.8° 且持续 0.4 秒 → 进入 LOCK 状态
```

---

## 📹 真实云台 vs 仿真模式

### 真实云台（带摄像头）

**你会看到：**
```
帧 1: [背景] [目标在右边]     [准心]
帧 2: [背景向左移] [目标]     [准心]  ← 云台向右转
帧 3: [背景继续左移] [目标]   [准心]
帧 4: [背景] [准心+目标重合]          ← 锁定！
```

**关键点：**
- ✅ 背景在移动（因为摄像头在转）
- ✅ 目标逐渐回到中心
- ✅ 准心始终在中心
- ✅ 你能"感觉到"云台在转动

---

### 仿真模式（无真实图像）

**你会看到：**
```
帧 1: [黑色背景] [目标框在右边] [准心]
帧 2: [黑色背景] [目标框]       [准心]  ← 看不出云台转了
帧 3: [黑色背景] [目标框]       [准心]
帧 4: [黑色背景] [准心+目标框]         ← 锁定！
```

**关键点：**
- ❌ 没有背景（黑色画面）
- ❌ 看不出云台在转动
- ✅ 但云台角度确实在变化（看右上角指示器）
- ✅ 误差在减小（看底部数值）

**如果云台在工作：**
- ✅ Yaw 滑块会左右移动
- ✅ Pitch 滑块会上下移动
- ✅ 角度数值在变化

---

### 方法 2：观察误差变化

底部显示：
```
时刻 1: Error: Y=+15.3 P=+8.2 deg  ← 目标偏离中心
时刻 2: Error: Y=+8.1 P=+4.5 deg   ← 云台转动，误差减小
时刻 3: Error: Y=+2.3 P=+1.1 deg   ← 继续减小
时刻 4: Error: Y=+0.5 P=+0.2 deg   ← 接近锁定
```

**如果云台在工作：**
- ✅ 误差数值在减小
- ✅ 最终接近 0°

---

### 方法 3：观察状态切换

左上角显示：
```
State: SEARCH  ← 灰色，寻找目标
State: TRACK   ← 黄色，开始跟踪
State: LOCK    ← 绿色，锁定目标（误差 < 0.8°）
State: LOST    ← 红色，丢失目标
```

**如果云台在工作：**
- ✅ 状态会从 SEARCH → TRACK → LOCK
- ✅ 颜色会变化

## 📊 当前测试结果分析

你刚才运行的结果：
```
Lock Rate: 0.0%        # 未达到锁定
Avg Error: 17.66 deg   # 平均误差 17.66 度
Switches: 2.0 /min     # 每分钟切换 2 次
```

**为什么 Lock Rate 是 0%？**

1. **目标移动太快**
   - 仿真目标速度：30 dps（每秒 30 度）
   - 云台最大速度：180 dps
   - 但 PID 需要时间响应

2. **LOCK 条件严格**
   - 需要误差 < 0.8°
   - 需要持续 0.4 秒
   - 目标一直在动，很难满足

3. **PID 参数可能需要调优**
   - 当前 Kp=5.0 可能偏小
   - 可以增大到 8.0 或 10.0

## 🎯 如何提高 Lock Rate？

### 方法 1：降低目标速度

编辑 `test_gimbal_visual.py`：
```python
scene.add_target(SimTarget(
    x=400, y=300, w=80, h=120,
    vx=10, vy=5,  # 从 30/20 降低到 10/5
    confidence=0.92, class_id="person"
))
```

### 方法 2：增大 PID 增益

编辑 `config.yaml`：
```yaml
controller:
  yaw_pid:
    kp: 8.0   # 从 5.0 增加到 8.0
    ki: 0.5   # 从 0.4 增加到 0.5
```

### 方法 3：放宽 LOCK 条件

编辑 `config.yaml`：
```yaml
controller:
  lock_error_threshold_deg: 1.5  # 从 0.8 放宽到 1.5
  lock_hold_time_s: 0.2          # 从 0.4 缩短到 0.2
```

## 总结

**准心不动是完全正常的！** 因为：

1. ✅ 准心 = 云台瞄准方向 = 画面中心

3. ✅ 云台转动时，整个画面都在转
4. ✅ 所以准心相对画面永远在中心
