## 🚀 使用方法

### 方法 1: REST API

**启动服务器:**
```bash
python scripts/run_api_server.py --host 0.0.0.0 --port 5000
```

**使用客户端:**
```python
from rws_tracking.api import TrackingClient

client = TrackingClient("http://localhost:5000")
client.start_tracking(camera_source=0)
status = client.get_status()
print(f"FPS: {status['fps']:.1f}")
client.stop_tracking()
```

**使用 cURL:**
```bash
curl http://localhost:5000/api/health
curl -X POST http://localhost:5000/api/start -H "Content-Type: application/json" -d '{"camera_source": 0}'
curl http://localhost:5000/api/status
```

### 方法 2: gRPC API

**启动服务器:**
```bash
python scripts/run_grpc_server.py --host 0.0.0.0 --port 50051
```

**使用客户端:**
```python
from rws_tracking.api import TrackingGrpcClient

with TrackingGrpcClient("localhost", 50051) as client:
    client.start_tracking(camera_source=0)

    # 实时流式更新
    for update in client.stream_status(update_rate_hz=10.0):
        print(f"FPS: {update['fps']:.1f}, Yaw: {update['gimbal']['yaw_deg']:.1f}°")
        if update['frame_count'] > 100:
            break

    client.stop_tracking()
```

**运行示例:**
```bash
python scripts/grpc_client_example.py
```

## 📊 API 对比

| 特性 | REST API | gRPC API |
|------|----------|----------|
| 协议 | HTTP/1.1 + JSON | HTTP/2 + Protobuf |
| 端口 | 5000 | 50051 |
| 延迟 | ~5-10ms | ~1-3ms |
| 流式传输 | ❌ | ✅ StreamStatus |
| 浏览器支持 | ✅ 原生 | ⚠️ 需要 grpc-web |
| 类型安全 | 运行时 | 编译时 |
| 带宽 | 较高（JSON） | 较低（二进制） |
| 调试 | 简单（curl） | 需要工具 |
| 多语言 | 手动实现 | 自动生成 |

## 🎯 API 端点总览

### REST API (8 个端点)
1. `GET /api/health` - 健康检查
2. `POST /api/start` - 启动跟踪
3. `POST /api/stop` - 停止跟踪
4. `GET /api/status` - 获取状态
5. `POST /api/gimbal/position` - 设置云台位置
6. `POST /api/gimbal/rate` - 设置云台速率
7. `GET /api/telemetry` - 获取遥测数据
8. `POST /api/config` - 更新配置

### gRPC API (9 个方法)
1. `HealthCheck` - 健康检查
2. `StartTracking` - 启动跟踪
3. `StopTracking` - 停止跟踪
4. `GetStatus` - 获取状态
5. `SetGimbalPosition` - 设置云台位置
6. `SetGimbalRate` - 设置云台速率
7. `GetTelemetry` - 获取遥测数据
8. `UpdateConfig` - 更新配置
9. `StreamStatus` - **实时流式更新** ⭐
