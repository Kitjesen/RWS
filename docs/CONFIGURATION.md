# RWS 配置字段详解

本文档详细说明 `config.yaml` 中所有配置字段的含义、单位、取值范围和调参建议。

## 配置文件结构

```yaml
camera:           # 相机内参和安装参数
detector:         # 目标检测器参数
selector:         # 目标选择器参数
controller:       # 云台控制器参数
driver_limits:    # 云台硬件限位
```

---

## 1. Camera (相机配置)

### 基本参数

#### `width`, `height`
- **含义**: 图像分辨率（像素）
- **类型**: `int`
- **默认值**: `1280`, `720`
- **取值范围**: 任意正整数
- **说明**: 必须与实际相机分辨率匹配

#### `fx`, `fy`
- **含义**: 相机焦距（像素）
- **类型**: `float`
- **默认值**: `970.0`, `965.0`
- **取值范围**: > 0
- **说明**:
  - 通过相机标定获得
  - 近似值：`fx ≈ width / (2 * tan(FOV_h/2))`
  - 例如：1280px 宽，60° 水平 FOV → fx ≈ 1110

#### `cx`, `cy`
- **含义**: 主点坐标（像素）
- **类型**: `float`
- **默认值**: `640.0`, `360.0`
- **取值范围**: `[0, width]`, `[0, height]`
- **说明**:
  - 理想情况下在图像中心
  - 通过相机标定获得精确值

### 畸变参数

#### `distortion_k1`, `distortion_k2`, `distortion_k3`
- **含义**: 径向畸变系数
- **类型**: `float`
- **默认值**: `0.0`
- **取值范围**: 通常 `[-1, 1]`
- **说明**:
  - OpenCV 5 参数模型
  - 广角镜头需要标定
  - 全部为 0 表示无畸变

#### `distortion_p1`, `distortion_p2`
- **含义**: 切向畸变系数
- **类型**: `float`
- **默认值**: `0.0`
- **取值范围**: 通常 `[-0.01, 0.01]`
- **说明**: 通常比径向畸变小得多

### 安装参数

#### `mount_roll_deg`, `mount_pitch_deg`, `mount_yaw_deg`
- **含义**: 相机相对云台的安装偏移角度（度）
- **类型**: `float`
- **默认值**: `0.0`
- **取值范围**: `[-180, 180]`
- **说明**:
  - 理想安装时全部为 0
  - 通过手眼标定获得
  - 小角度近似（< 10°）

## 2. Detector (检测器配置)

#### `model_path`
- **含义**: YOLO 模型文件路径
- **类型**: `str`
- **默认值**: `"yolo11n.pt"`
- **说明**:
  - 支持 YOLO11n, YOLO11s, YOLO11m 等
  - 可以是本地路径或 Ultralytics 模型名

#### `confidence_threshold`
- **含义**: 检测置信度阈值
- **类型**: `float`
- **默认值**: `0.25`
- **取值范围**: `[0.0, 1.0]`
- **调参建议**:
  - 提高（0.4-0.6）：减少误检，但可能漏检
  - 降低（0.1-0.2）：增加召回率，但误检增多

#### `nms_iou_threshold`
- **含义**: NMS (非极大值抑制) IoU 阈值
- **类型**: `float`
- **默认值**: `0.7`
- **取值范围**: `[0.0, 1.0]`
- **调参建议**:
  - 提高（0.8-0.9）：保留更多重叠框
  - 降低（0.4-0.6）：更激进地去除重叠

#### `tracker`
- **含义**: BoT-SORT 配置文件
- **类型**: `str`
- **默认值**: `"botsort.yaml"`
- **说明**: Ultralytics 跟踪器配置

#### `class_whitelist`
- **含义**: 类别白名单（只检测这些类别）
- **类型**: `List[str]`
- **默认值**: `["person"]`
- **说明**:
  - COCO 80 类：person, car, truck, bicycle, etc.
  - 空列表 = 检测所有类别

#### `device`
- **含义**: 推理设备
- **类型**: `str`
- **默认值**: `"cuda:0"`
- **选项**: `"cuda:0"`, `"cpu"`, `"mps"` (Mac)

#### `img_size`
- **含义**: YOLO 输入尺寸
- **类型**: `int`
- **默认值**: `640`
- **取值范围**: `[320, 1280]`（32 的倍数）
- **调参建议**:
  - 增大（960, 1280）：精度提高，速度降低
  - 减小（416, 320）：速度提高，精度降低

## 3. Selector (选择器配置)

### 权重配置

#### `weights.confidence`
- **含义**: 置信度权重
- **类型**: `float`
- **默认值**: `0.35`
- **取值范围**: `[0.0, 1.0]`
- **说明**: 高置信度目标得分更高

#### `weights.size`
- **含义**: 尺寸权重
- **类型**: `float`
- **默认值**: `0.20`
- **取值范围**: `[0.0, 1.0]`
- **说明**: 大目标得分更高

#### `weights.center_proximity`
- **含义**: 中心距离权重
- **类型**: `float`
- **默认值**: `0.20`
- **取值范围**: `[0.0, 1.0]`
- **说明**: 靠近画面中心的目标得分更高

#### `weights.track_age`
- **含义**: 跟踪年龄权重
- **类型**: `float`
- **默认值**: `0.15`
- **取值范围**: `[0.0, 1.0]`
- **说明**: 跟踪时间长的目标得分更高（更稳定）

#### `weights.class_weight`
- **含义**: 类别权重
- **类型**: `float`
- **默认值**: `0.10`
- **取值范围**: `[0.0, 1.0]`
- **说明**: 配合 `preferred_classes` 使用

#### `weights.switch_penalty`
- **含义**: 切换惩罚
- **类型**: `float`
- **默认值**: `0.30`
- **取值范围**: `[0.0, 1.0]`
- **说明**:
  - 切换到新目标时的分数惩罚
  - 增大可减少频繁切换

### 行为参数

#### `min_hold_time_s`
- **含义**: 最小保持时间（秒）
- **类型**: `float`
- **默认值**: `0.4`
- **取值范围**: `[0.0, 2.0]`
- **调参建议**:
  - 增大（0.6-1.0）：减少切换，但响应变慢
  - 减小（0.2-0.3）：快速响应，但可能抖动

#### `delta_threshold`
- **含义**: 切换阈值（分数差）
- **类型**: `float`
- **默认值**: `0.12`
- **取值范围**: `[0.0, 0.5]`
- **说明**:
  - 新目标分数必须高出当前目标此值才切换
  - 增大可减少切换

#### `preferred_classes`
- **含义**: 类别偏好权重
- **类型**: `Dict[str, float]`
- **默认值**: `{"person": 1.0}`
- **示例**:
  ```yaml
  preferred_classes:
    person: 1.0      # 最高优先级
    car: 0.6         # 中等优先级
    truck: 0.4       # 较低优先级
  ```

#### `age_norm_frames`
- **含义**: 年龄归一化帧数
- **类型**: `int`
- **默认值**: `60`
- **说明**: 跟踪 60 帧后年龄分数达到最大值

## 4. Controller (控制器配置)

### PID 参数

#### `yaw_pid.kp`, `pitch_pid.kp`
- **含义**: 比例增益
- **类型**: `float`
- **默认值**: `5.0`, `5.5`
- **取值范围**: `[0.0, 50.0]`
- **调参建议**:
  - **增大**: 响应更快，但可能震荡
  - **减小**: 更平滑，但响应慢
  - **典型值**: 5-15

#### `yaw_pid.ki`, `pitch_pid.ki`
- **含义**: 积分增益
- **类型**: `float`
- **默认值**: `0.4`, `0.35`
- **取值范围**: `[0.0, 5.0]`
- **调参建议**:
  - **增大**: 消除稳态误差更快，但可能超调
  - **减小**: 更稳定，但稳态误差大
  - **典型值**: 0.2-1.0

#### `yaw_pid.kd`, `pitch_pid.kd`
- **含义**: 微分增益
- **类型**: `float`
- **默认值**: `0.35`, `0.35`
- **取值范围**: `[0.0, 5.0]`
- **调参建议**:
  - **增大**: 阻尼增强，减少超调
  - **减小**: 响应更快，但可能震荡
  - **典型值**: 0.2-0.8

#### `yaw_pid.integral_limit`
- **含义**: 积分限幅
- **类型**: `float`
- **默认值**: `40.0`
- **单位**: 度
- **说明**: 防止积分饱和

#### `yaw_pid.output_limit`
- **含义**: 输出限幅
- **类型**: `float`
- **默认值**: `180.0`
- **单位**: 度/秒
- **说明**: 最大云台速率

#### `yaw_pid.derivative_lpf_alpha`
- **含义**: 微分低通滤波系数
- **类型**: `float`
- **默认值**: `0.4`
- **取值范围**: `[0.0, 1.0]`
- **说明**:
  - 0 = 无滤波（噪声大）
  - 1 = 完全滤波（延迟大）
  - 典型值: 0.3-0.5

#### `yaw_pid.feedforward_kv`
- **含义**: 速度前馈系数
- **类型**: `float`
- **默认值**: `0.75`, `0.70`
- **取值范围**: `[0.0, 2.0]`
- **说明**:
  - 补偿目标运动
  - 1.0 = 完全补偿
  - 典型值: 0.5-1.0

### 状态机参数

#### `lock_error_threshold_deg`
- **含义**: LOCK 状态误差阈值（度）
- **类型**: `float`
- **默认值**: `0.8`
- **取值范围**: `[0.1, 5.0]`
- **调参建议**:
  - **增大（1.5-3.0）**: 更容易进入 LOCK，Lock Rate 提高
  - **减小（0.3-0.5）**: 更严格的锁定，精度更高

#### `lock_hold_time_s`
- **含义**: LOCK 状态保持时间（秒）
- **类型**: `float`
- **默认值**: `0.4`
- **取值范围**: `[0.1, 2.0]`
- **说明**: 误差必须持续低于阈值此时间才进入 LOCK

#### `max_rate_dps`
- **含义**: 最大云台速率（度/秒）
- **类型**: `float`
- **默认值**: `180.0`
- **取值范围**: `[10.0, 500.0]`
- **说明**: 硬件限制

### 补偿参数

#### `latency_compensation_s`
- **含义**: 延迟补偿时间（秒）
- **类型**: `float`
- **默认值**: `0.033`
- **取值范围**: `[0.0, 0.2]`
- **说明**:
  - 根据目标速度外推位置
  - 典型值: 1-2 帧时间

#### `command_lpf_alpha`
- **含义**: 命令低通滤波系数
- **类型**: `float`
- **默认值**: `0.75`
- **取值范围**: `[0.0, 1.0]`
- **说明**:
  - 平滑输出命令
  - 增大可减少抖动

### 扫描模式参数

#### `scan_freq_hz`
- **含义**: 扫描频率（Hz）
- **类型**: `float`
- **默认值**: `0.15`
- **说明**: SEARCH 状态下的扫描频率

#### `scan_yaw_scale`, `scan_pitch_scale`
- **含义**: 扫描幅度（度）
- **类型**: `float`
- **默认值**: `30.0`, `15.0`

### 弹道补偿 (可选)

#### `ballistic.enabled`
- **含义**: 是否启用弹道补偿
- **类型**: `bool`
- **默认值**: `false`

#### `ballistic.model_type`
- **含义**: 补偿模型类型
- **类型**: `str`
- **选项**: `"simple"`, `"table"`
- **说明**:
  - `simple`: 二次函数模型
  - `table`: 查找表插值

#### `ballistic.target_height_m`
- **含义**: 目标高度（米）
- **类型**: `float`
- **默认值**: `1.8`
- **说明**: 用于估算距离

#### `ballistic.quadratic_a`, `quadratic_b`, `quadratic_c`
- **含义**: 二次函数系数
- **类型**: `float`
- **说明**: `compensation = a*d² + b*d + c`

### 自适应 PID (可选)

#### `adaptive_pid.enabled`
- **含义**: 是否启用自适应 PID
- **类型**: `bool`
- **默认值**: `false`

#### `adaptive_pid.scheduler_type`
- **含义**: 调度器类型
- **类型**: `str`
- **选项**: `"error_based"`, `"distance_based"`

#### `adaptive_pid.low_error_threshold_deg`
- **含义**: 低误差阈值（度）
- **类型**: `float`
- **默认值**: `2.0`

#### `adaptive_pid.high_error_threshold_deg`
- **含义**: 高误差阈值（度）
- **类型**: `float`
- **默认值**: `10.0`

#### `adaptive_pid.low_error_multiplier`
- **含义**: 低误差增益倍数
- **类型**: `float`
- **默认值**: `0.6`
- **说明**: 小误差时降低增益，精确锁定

#### `adaptive_pid.high_error_multiplier`
- **含义**: 高误差增益倍数
- **类型**: `float`
- **默认值**: `1.5`
- **说明**: 大误差时提高增益，快速追赶

## 5. Driver Limits (云台限位)

#### `yaw_min_deg`, `yaw_max_deg`
- **含义**: Yaw 轴限位（度）
- **类型**: `float`
- **默认值**: `-160.0`, `160.0`
- **说明**: 根据硬件规格设置

#### `pitch_min_deg`, `pitch_max_deg`
- **含义**: Pitch 轴限位（度）
- **类型**: `float`
- **默认值**: `-45.0`, `75.0`

#### `max_rate_dps`
- **含义**: 最大速率（度/秒）
- **类型**: `float`
- **默认值**: `240.0`

#### `deadband_dps`
- **含义**: 死区（度/秒）
- **类型**: `float`
- **默认值**: `0.2`
- **说明**: 小于此值的命令被忽略

#### `inertia_time_constant_s`
- **含义**: 惯性时间常数（秒）
- **类型**: `float`
- **默认值**: `0.05`
- **说明**: 仿真用，真实云台由硬件决定

#### `static_friction_dps`
- **含义**: 静摩擦阈值（度/秒）
- **类型**: `float`
- **默认值**: `0.5`

#### `coulomb_friction_dps`
- **含义**: 库仑摩擦（度/秒）
- **类型**: `float`
- **默认值**: `2.0`

## 调参流程

### 1. 基础调参（PID）

```yaml
# 第一步：只调 Kp
controller:
  yaw_pid:
    kp: 5.0   # 逐步增大直到出现轻微震荡
    ki: 0.0   # 暂时关闭
    kd: 0.0

# 第二步：加入 Ki
controller:
  yaw_pid:
    kp: 8.0   # 上一步的值
    ki: 0.3   # 逐步增大直到稳态误差消除
    kd: 0.0

# 第三步：加入 Kd
controller:
  yaw_pid:
    kp: 8.0
    ki: 0.3
    kd: 0.4   # 逐步增大直到震荡消除
```

### 2. 提高 Lock Rate

```yaml
# 方法 1：放宽 LOCK 条件
controller:
  lock_error_threshold_deg: 1.5  # 从 0.8 增加到 1.5
  lock_hold_time_s: 0.2          # 从 0.4 减少到 0.2

# 方法 2：增大 PID 增益
controller:
  yaw_pid:
    kp: 10.0  # 增大 Kp
    ki: 0.5

# 方法 3：启用自适应 PID
controller:
  adaptive_pid:
    enabled: true
    scheduler_type: "error_based"
```

### 3. 减少目标切换

```yaml
selector:
  min_hold_time_s: 0.6      # 增大保持时间
  delta_threshold: 0.15     # 增大切换阈值
  weights:
    switch_penalty: 0.5     # 增大切换惩罚
```

### 4. 减少震荡

```yaml
controller:
  yaw_pid:
    kp: 5.0                      # 减小 Kp
    derivative_lpf_alpha: 0.5    # 增大微分滤波
  command_lpf_alpha: 0.8         # 增大命令滤波
```

## 性能指标参考

### Lock Rate
- **优秀**: > 80%
- **良好**: 50-80%
- **可接受**: 20-50%
- **需改进**: < 20%

### Avg Error
- **优秀**: < 0.5°
- **良好**: 0.5-1.5°
- **可接受**: 1.5-3.0°
- **需改进**: > 3.0°

### Switches
- **优秀**: < 2 /min
- **良好**: 2-5 /min
- **可接受**: 5-10 /min
- **需改进**: > 10 /min

## 常见问题

### Q: 如何快速找到合适的 PID 参数？
A: 使用自动调优工具：
```python
from src.rws_tracking.tools.tuning import grid_search_pid
tuned_cfg, best_metrics = grid_search_pid(base_cfg, cam)
```

### Q: Lock Rate 始终为 0%？
A:
1. 检查目标移动速度（仿真中降低速度）
2. 放宽 `lock_error_threshold_deg` 到 2-3°
3. 增大 Kp 到 10-15

### Q: 云台震荡？
A:
1. 减小 Kp
2. 增大 `command_lpf_alpha` 到 0.8
3. 增大 `derivative_lpf_alpha` 到 0.6

### Q: 目标频繁切换？
A:
1. 增大 `min_hold_time_s` 到 0.6-1.0
2. 增大 `delta_threshold` 到 0.15-0.20
3. 增大 `switch_penalty` 到 0.5
